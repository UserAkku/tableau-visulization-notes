INTRODUCTION // PDF-09
Tableau Kya Hai? — Poori Kahani 🎯
Business Intelligence ka superstar — raw data ko decisions mein convert karne ki machine
◈
Real-World Analogy Se Samjho
CONCEPT

Imagine karo — ek company ka CFO hai. Uske paas ek Excel file hai jisme 3 lakh rows hain. Sales, returns, customer names, regions, product categories — sab kuch. Har mahine end pe woh 5 ghante lagata hai pivot tables banane mein, graphs copy-paste karne mein, phir ek PowerPoint mein dalne mein. Aur jab boss ne ek naya question pucha — toh process fir se shuru!

Tableau ne yahi problem solve ki. Ab woh CFO sirf ek baar data connect karta hai, kuch cheezein drag karta hai — aur interactive dashboard ready! Boss koi bhi filter lagaye, chart real-time update ho jaata hai. Koi manual work nahi.

💡BI (Business Intelligence) kya hoti hai? Data ko sirf store karna BI nahi hai. BI matlab hai data ko dekh ke smart decisions lena. "Q3 mein North region mein sales kyun giri?" ya "Kaunsa product category sabse zyada discount pe bik raha hai aur phir bhi loss mein hai?" — in questions ka jawab dhundhna BI hai. Tableau ye kaam minutes mein karta hai jo Excel mein ghante leta tha.
EXCEL KI PROBLEM
1 lakh+ rows pe slow ho jaata hai. Ek analysis fir se karni ho toh sab manually repeat karo. Sharing ke liye email attachment bhejni padti hai. Formulas complex ho jaate hain. Version control ka koi concept nahi.
TABLEAU KA SOLUTION
Millions of rows instantly handle karta hai. Ek baar dashboard banao, hamesha ke liye ready. Share karo — dusra insaan apne browser mein khol sakta hai. Live data connect ho sakti hai. AI insights automatically deta hai (Tableau Pulse).
KAB USE KARO?
Jab data frequently change hota ho. Jab multiple logon ko same analysis dekhni ho. Jab interactive exploration chahiye — "zoom in" karna ho specific region ya product pe. Jab professional presentations banani ho.
◈

INTERFACE // PDF-09
Tableau Interface Ka Poora Tour 🗺️
Pehli baar khola toh scary lagta hai — har cheez ka naam aur kaam ek baar mein samjho
◈
Start Screen — Tableau Kholne Pe Kya Dikhta Hai?
BASICS

Tableau Desktop kholte hi teen panels dikhte hain — Connect, Open, Discover. Ek ek samjhte hain:

CONNECT PANE
Naya data connect karo. Yahan se tum Excel file, CSV, Google Sheets, SQL database, ya cloud sources connect kar sakte ho. "To a File" section mein Excel/CSV hogi. "To a Server" mein databases honge (MySQL, PostgreSQL etc.).

Ek baar connect karo → data Tableau mein aata hai → analysis shuru!
OPEN PANE
Pehle se saved kaam open karo. Tableau files .twb (workbook without data) ya .twbx (workbook with data packaged) format mein save hoti hain.

Recently opened files yahan shortcut pe milti hain. Training files bhi yahan sample workbooks mein milte hain.
DISCOVER PANE
Practice ke liye sample data. Tableau ke saath built-in datasets aate hain — Superstore Sales, World Indicators etc. Inhe use karke practice karo bina apna data dhundhne ke.

Superstore dataset — is course mein isi pe saara kaam hoga!
◈
Main Workspace — 6 Areas Jo Roz Use Honge
WORKSPACE

Data connect karne ke baad worksheet open hoti hai. Ye wo jagah hai jahan actual analysis hoti hai. 6 main areas hain — inhein yaad kar lo!

①
DATA PANE (Left Side) — Tumhara Data Yahan Hai
Connected data source ke saare columns yahan dikhte hain. Blue pills = Dimensions (text/category data), Green pills = Measures (numeric data). Yahan se fields ko drag karke analysis shuru hoti hai.

Important: Tableau automatically decide karta hai kaunsa field Dimension hai aur kaunsa Measure — numbers green, baaki blue. Galat categorize ho toh right-click → Convert to Dimension/Measure.
②
COLUMNS & ROWS SHELVES (Top) — X aur Y Axis
Columns shelf = X-axis (horizontal). Rows shelf = Y-axis (vertical). Fields yahan drop karo aur chart automatically banna shuru ho jaata hai.

Example: "Region" Rows pe drop karo + "SUM(Sales)" Columns pe → automatic horizontal bar chart! Ye Tableau ka core magic hai.
③
MARKS CARD (Left Center) — Chart Ka Look Control
Chart pe sirf 2 variables (X aur Y) nahi — aur bhi information encode kar sakte ho! Color, Size, Shape, Label, Tooltip yahan se control hote hain.

Example: Sales vs Profit ka scatter plot hai. Ab "Category" ko Color pe drag karo — ab har category ka alag color! Ye ek chart mein 3rd dimension add ho gayi bina axes change kiye.
④
CANVAS / VIEW (Center) — Yahan Chart Banta Hai
Ye empty white area jahan tumhara chart appear hoga. Jaise jaise Rows/Columns pe fields drop karo, canvas pe chart banta jaata hai. Tableau real-time mein update karta rehta hai.

Canvas pe right-click karo → various options milti hain jaise annotations add karna, trend lines add karna, reference lines add karna.
⑤
SHOW ME PANEL (Top Right) — Shortcut Chart Types
Ctrl+Click se Data Pane mein multiple fields select karo, phir Show Me button click karo (top right corner). Tableau automatically best chart types highlight karega tumhare selected data ke liye.

Gray = ye chart type is data ke saath nahi bnega. Highlighted = ye possible hain. Click karo → chart type switch ho jaata hai instantly!
⑥
FILTERS SHELF — Kya Dikhana Hai Control Karo
Koi bhi field Filters shelf pe drag karo → chart mein sirf woh data dikhega jo condition satisfy kare. Example: "Category" Filters pe drag karo → dialog box aayega → sirf "Technology" select karo → chart sirf Technology ka data dikhayega.

Filters ko dashboard pe show bhi kar sakte ho taaki viewers khud filter kar sakein!

CONNECTION TYPE // PDF-09
Live Connection vs Extract — Ek Zaruri Decision 🔌
Data connect karne ke 2 tarike — dono mein se galat choose kiya toh performance suffer karegi
◈
Dono Ko Depth Mein Samjho
CORE CONCEPT

Jab Tableau ko data source se connect karte ho, toh Tableau poochhta hai — "Data ko real-time use karna hai ya ek copy bana loon?" Ye choice bahut important hai aur use case pe depend karti hai.

🔴 LIVE CONNECTION — Real Time

Har query seedha original database pe jaati hai. Koi copy nahi banti Tableau ke paas.

Kaise kaam karta hai:

  • Tum ek chart pe filter lagate ho
  • Tableau ek SQL query generate karta hai
  • Query database pe jaati hai
  • Database result return karta hai
  • Tableau chart update karta hai

Fayde: Data hamesha latest — agar koi naya order aaya toh turant dikhai dega.

Nuksan: Speed database ki speed pe depend karti hai. Slow database = Slow Tableau. Heavy queries pe lag ho sakta hai.

BEST FOR: Real-time ops dashboards, live inventory, today's sales

🟢 EXTRACT (.hyper file) — Lightning Fast

Tableau data ki ek optimized copy apne paas save kar leta hai — .hyper format mein (in-memory columnar store).

Kaise kaam karta hai:

  • Ek baar extract create hoti hai (data copy hoti hai)
  • Ab har query .hyper file pe jaati hai — database pe nahi
  • .hyper columnar format mein hoti hai — visual queries ke liye optimized
  • Offline bhi kaam karta hai!

Fayde: 10x faster queries. Internet nahi chahiye. Heavy workbooks share karo clients ke saath.

Nuksan: Data stale ho sakta hai — regularly refresh karna padta hai.

BEST FOR: Reports, client sharing, yesterday's data analysis

🔑Decision Rule: Data baar baar change hota hai aur decisions real-time pe depend karte hain? → Live Connection. Reports banana hai, weekly/monthly analysis hai, ya database slow hai? → Extract. Extract schedule bhi kar sakte ho — har raat 2 baje auto-refresh!

CORE CONCEPT // PDF-09
Dimensions vs Measures — Tableau Ka DNA 🔵🟢
Ye concept samajhna mandatory hai — yahan ghalti hui toh saari analysis galat hogi
◈
Ek Simple Question Se Decide Karo
TRICK

Koi bhi field dekho aur ek simple sawaal pucho: "Kya main is field ka average ya total nikal sakta hoon? Kya ye ek number hai?"

  • Agar haan — ye ek Measure (Green) hai. Example: Sales, Profit, Quantity, Age
  • Agar nahi — ye ek Dimension (Blue) hai. Example: Customer Name, City, Category, Order ID
🤔Wait — "Order ID" toh number hai, toh Measure hona chahiye? Nahi! Order ID ek identifier hai — uska average nikalna senseless hai. Isliye ye Dimension hai. Tableau kabhi kabhi galat categorize karta hai — right-click → Convert to Dimension isse fix karo!
◈
Dono Ka Behavior Bilkul Alag Hai
BEHAVIOR
🔵 DIMENSIONS — "WHO / WHAT / WHERE"

Categorize aur slice karte hain data ko. Ye descriptive information hoti hai.

Examples from Superstore:

  • Customer Name — Darrin Van Huff, John Smith
  • Category — Furniture, Technology, Office Supplies
  • Region — East, West, Central, South
  • State — California, Texas, New York
  • Order Date — Dates (ye bhi blue hoti hai)

Chart mein kya karta hai: Rows/Columns pe rakho toh data groups mein divide hota hai. "East", "West", "Central" — teen alag bars honge.

🔵 Category🔵 State🔵 Customer Name
🟢 MEASURES — "HOW MUCH / HOW MANY"

Quantify karte hain — aggregate ho sakte hain (SUM, AVG, COUNT, MIN, MAX).

Examples from Superstore:

  • Sales — $226 average per order
  • Profit — Can be negative!
  • Quantity — How many items ordered
  • Discount — Percentage discount given

Chart mein kya karta hai: Automatically aggregate hoti hai. Jab tum Sales ko Y-axis pe daalo, Tableau automatically SUM(Sales) compute karta hai. Default aggregation change bhi kar sakte ho — right-click → Measure → Average/Count etc.

🟢 SUM(Sales)🟢 AVG(Profit)🟢 COUNT(Orders)
PRACTICAL EXAMPLE — EK CHART BANANA
Goal: "Har Region mein Total Sales kitni hai?" — Bar chart banana hai.
1
Region (Dimension — Blue) → Rows shelf pe drag karo
Ab chart mein 4 rows ban gayi — East, West, Central, South. Ye categories hain.
2
Sales (Measure — Green) → Columns shelf pe drag karo
Tableau automatically SUM(Sales) compute karega har region ke liye. Bar chart ready!
3
Result
4 bars — har region ki total sales. Itna hi simple hai Tableau!

Discrete vs Continuous — Chart Ka Look Badlo ⚙️
Dimension/Measure = data ki nature. Discrete/Continuous = Tableau ko visual instruction. Dono alag cheezein hain!
◈
Date Field Ka Example — Sabse Clear Explanation
KEY EXAMPLE

Ye concept Date field se best samajhta hai. "Order Date" ek Dimension hai (blue). Lekin ise Discrete ya Continuous dono tarike se use kar sakte ho — aur result bilkul alag hoga:

DATE DISCRETE → BAR CHART (Alag Headers)

Har month ek separate labeled category ban jaati hai.

Output: "Jan 2023" | "Feb 2023" | "Mar 2023" — teen alag bars.

Pill color: 🔵 MONTH(Order Date)

Use karo jab: Specific months compare karne hoon (Jan better tha ya Feb?)

DATE CONTINUOUS → LINE CHART (Flowing Axis)

Date ek continuous flowing axis ban jaati hai.

Output: Jan se Dec tak ek smooth line — trend dikhata hai.

Pill color: 🟢 MONTH(Order Date)

Use karo jab: Time pe trend dekhna ho (sales grow ho rahi hai ya gir rahi hai?)

⚡Toggle karna easy hai: Date pill pe right-click karo → "Discrete" ya "Continuous" select karo. Tableau instantly chart type switch kar deta hai! Experiment karo — dono try karo aur dekho kaunsa better insight deta hai.

VISUALIZATION // PDF-09+10
Charts Banana — Visual Vocabulary Poori Seekho 📊
Kaunsa chart kab use karo — ye decision analytics mein sabse critical hai
◈
3 Tarike Chart Banana — Kab Kaunsa Use Karo?
METHODS
1
Drag & Drop — Standard Method (Sabse Zyada Use)
Data Pane se Dimensions aur Measures ko Rows/Columns shelf pe manually drag karo. Ye method best hai jab exact control chahiye — exactly kaunsa field X pe aur kaunsa Y pe.

Process: Dimension → Rows ya Columns. Measure → Rows ya Columns (opposite side pe). Tableau auto-select karta hai chart type based on data types.

Advanced: Ek se zyada Dimensions ek shelf pe rakh sakte ho — Tableau grouped/nested view create karega!
2
Show Me Panel — Explore Mode (Naya Experiment Karo)
Ctrl+Click (Mac: Cmd+Click) se Data Pane mein multiple fields select karo simultaneously, phir top-right "Show Me" button click karo.

Tableau highlighted chart types dikhayega jo is data combination ke saath possible hain. Gray wale possible nahi hain. Best use case: Jab pata nahi kaunsa chart best hoga — Show Me suggest karega!

Click karo kisi bhi highlighted chart type pe → instantly switch ho jaata hai. Explore karo!
3
Double Click — Fast Method (Quick Exploration)
Data Pane se directly kisi field pe double-click karo. Tableau khud decide karta hai kahan rakhna hai aur kaunsa chart type best hai.

Best for: Quickly explore karna — "ye field interesting hai, dekh lete hain chart mein kaisi lagti hai." Production dashboards ke liye drag-and-drop prefer karo for control.

MARKS CARD // PDF-10
Marks Card — Ek Chart Mein 5 Variables Encode Karo 🎨
Sirf X aur Y se zyada — Color, Size, Shape, Label, Tooltip se data aur bhi powerful ho jaata hai
◈
Marks Card Ko Depth Mein Samjho
DETAIL

Marks Card Tableau ka secret weapon hai. Iske bina tum sirf 2D charts bana sakte ho. Iske saath tum ek hi chart mein 4-5 variables encode kar sakte ho! Har option ek naya dimension add karta hai without changing axes.

🎨 COLOR — 3rd Variable Add Karo

Koi Dimension drag karo → Har category ka alag color milega (qualitative).

Koi Measure drag karo → Gradient color scale milegi — low value = light color, high value = dark color (sequential).

Example: Sales vs Profit scatter plot pe "Category" ko Color pe drag karo → ab Furniture ka alag color, Technology ka alag, Office Supplies ka alag. Ek hi chart mein 3 variables!

💡Color blind users ke liye Tableau mein color blind friendly palettes available hain — accessibility ke liye zaroor use karo professional dashboards mein!
📏 SIZE — 4th Variable Add Karo

Koi Measure drag karo → Data value ke hisaab se points/bars ka size change hoga.

Example: Sales vs Profit scatter plot mein "Quantity" ko Size pe drag karo → zyada quantity = bada point. Ab ek chart mein 4 variables: Sales (X), Profit (Y), Category (Color), Quantity (Size)!

Bubble Charts: Yahi concept bubble chart mein use hota hai — size ek important metric ko represent karta hai.

⚠️Too many encodings = cluttered chart! Max 3-4 variables encode karo ek chart mein. Clarity > Complexity.
🏷️ LABEL — Values Directly Dikhao

Measure drag karo → Har data point pe actual value directly dikhai degi chart ke upar.

Example: Bar chart mein SUM(Sales) ko Label pe drag karo → har bar ke upar exact number dikhega.

Label pe right-click → Format karo — font size, color, position sab customize hota hai.

💬 TOOLTIP — Hover Info Customize Karo

Mouse hover karne pe popup aata hai. By default kuch info show karta hai. Aur bhi fields add kar sakte ho — Tooltip shelf pe drag karo.

Pro tip: Marks → Edit Tooltip click karo → paragraph likhne jaisi flexibility milti hai — text, fields, aur bhi add karo!

Example: "Customer: [Customer Name] ordered [Quantity] units with Profit: [Profit]"

🔷 SHAPE — Scatter Plots Mein Differentiate

Scatter plots mein Dimension ko Shape pe drag karo → har category ka alag shape (circle, square, triangle, cross etc.)

Color ke saath combine karo for maximum differentiation — important when printing in B&W!

🔍 DETAIL — Granularity Increase Karo

Koi Dimension Detail pe drag karo → chart mein aur specific breakdown aajayegi without changing visual encoding (no new color/size/shape).

Example: Region level pe chart hai, "State" ko Detail pe drop karo → data state level pe breakdown hoga but visually same lagega.

MARK TYPE SELECTOR — CHART TYPE MANUALLY CHANGE KARO

Marks Card ke top pe ek dropdown hai — Mark Type Selector. Yahan se manually chart type change kar sakte ho: Automatic, Bar, Line, Area, Square, Circle, Shape, Text, Map, Pie, Gantt, Polygon.

🎯Important: Jab "Automatic" select hota hai, Tableau khud decide karta hai best mark type. Jab tum manually change karte ho — woh fixed ho jaata hai. "Text" select karo → scatter plot ki jagah actual text values dikhengi — text tables banana ke liye!

DATA MODELING // PDF-10
Data Joining — Multiple Tables Ko Ek Saath Use Karo 🔗
Real world mein data kabhi ek table mein nahi hota — Orders, Customers, Products, Returns — sab alag alag
◈
Real World Problem — Kyun Join Chahiye?
WHY

Superstore dataset mein typically 3 files hote hain: Orders (transactions), People (regional managers), Returns (returned orders). Agar tum yeh jaanna chahte ho ki "Kaunse regional manager ke area mein returns zyada hain?" — toh Orders aur People aur Returns teen ko combine karna padega. Yahi join hai.

🔑Join ke liye ek common key field dono tables mein hona chahiye — jaise "Order ID" Orders table aur Returns table dono mein hai. Isi pe join hoga.
◈
4 Ways Tables Combine Karne Ke — Poori Detail
TYPES
J
JOINS — Row-by-Row Physical Merge
Do tables ko ek common key field pe physically merge karo. Ek combined table banti hai.

4 Types of Joins:
JOIN TYPEKAUNSI ROWS AATI HAIN?KAB USE KARO?
INNER JOINSirf woh rows jo DONO tables mein match karti hainJab sirf matching records chahiye — default choice
LEFT JOINLeft table ki SAARI rows + matching right table rows. Non-matching right rows = NULLJab left table sab rakho par right se extra info add karni ho — most common!
RIGHT JOINRight table ki SAARI rows + matching left table rowsLeft join ki mirror image — less common
FULL OUTER JOINDONO tables ki saari rows — matching + non-matching (NULLs fill hote hain)Jab koi data miss nahi karna — audit use cases
⚠️Join ka trap: Join se duplicate rows aa sakti hain! Agar Orders table mein ek customer ka 5 orders hain aur Customers table mein ek row, toh LEFT JOIN ke baad customer info 5 baar repeat hogi. Ye expected behavior hai — but aggregation galat ho sakti hai. Isliye Relationships prefer karo Joins ke bajaye!
U
UNION — Vertically Stack Karo (Same Structure)
Data ko upar-neeche stack karo. Jaise January ka data + February ka data ek saath — same columns, alag rows.

Strict Rule: Dono tables ke columns IDENTICAL hone chahiye (same names, same data types). Warna Tableau null values se fill karega.

How to do in Tableau: Data source page pe base table drag karo → doosri table ko base table ke neeche overlap karke drop karo → Union dialog aayega.

Wildcard Union: Tableau ek powerful feature deta hai — agar tumhare paas multiple files hain same structure ke (Jan_2023.csv, Feb_2023.csv, Mar_2023.csv), toh wildcard pattern (*.csv) se automatically sab ko union kar sakte ho!
R
RELATIONSHIPS — Smart Modern Approach (Recommended!)
Relationships Tableau 2020.2 mein introduce hua — ye Join se better aur smarter hai. Tables physically merge nahi hoti — alag rehti hain. Tableau dynamically analysis ke time appropriate join lagata hai.

Kyun better hai?
  • No Duplicate Problem: Aggregation hamesha correct hoti hai, duplicates nahi aate
  • Different Granularity Handle: Orders table (transaction level) + Customers table (customer level) — different granularity, Relationship handle kar leti hai
  • Performance: Join only hota hai jab zaroorat hoti hai — unnecessary data process nahi hota
  • Flexible: Context ke hisaab se automatically sahi join type use karta hai
🏆Best Practice: Jab bhi possible ho — Relationship use karo, Join nahi. Join sirf tab jab Relationship kaam nahi kar sakti ya legacy workbooks ke saath.
B
DATA BLENDING — Alag Sources Ko Combine Karo (Last Resort)
Blending tab use karo jab data bilkul alag sources se aata ho — jaise ek Excel file + ek SQL database.

Kaise kaam karta hai:
Primary source set karo. Secondary source connect karo. Ek common "link field" select karo. Tableau secondary source ko aggregate karta hai, phir primary se join karta hai. Data kabhi physically merge nahi hota — dono sources alag rehte hain.

Limitations: Slow ho sakta hai. Limited functionality compared to native joins. Left join hi possible hai (primary = left). Isliye last resort option hai — agar Join ya Relationship possible hai toh Blend mat karo.

DATA ORGANIZATION // PDF-10
Groups, Bins, Hierarchies & Sets — Data Ko Organize Karo 📁
Raw data ko meaningful categories mein organize karne ke powerful tools — bina original data change kiye
◈
Groups — Categories Ko Merge Karo
GROUPS

Problem: Superstore mein 50 states hain. Analysis ke liye "East Zone" (NY, NJ, CT), "West Zone" (CA, OR, WA), "South Zone" etc. banana chahte ho. Groups se ye possible hai!

1
State field pe right-click karo Data Pane mein
→ Group → By Field select karo
2
States select karo jo ek group mein jaane chahiye
Ctrl+Click se multiple select karo → "Group" button pe click karo → naam do (e.g., "East Zone")
3
Repeat for other groups
Naya field banta hai Data Pane mein — "State (group)" — ye use karo analysis mein!
💡Chart pe directly bhi group ban sakta hai: Chart mein data points select karo (Ctrl+Click) → right-click → Group! Visually select karke groups banana bahut fast hai.
◈
Bins — Numbers Ko Ranges Mein Divide Karo (Histograms!)
BINS

Problem: Sales field mein values hain 0 se 14000 tak. Ye sab alag alag values hain — bar chart mein 3000+ bars bante! Bins in values ko ranges mein group karta hai — "0-500", "500-1000", "1000-2000" etc.

1
Measure pe right-click karo (e.g., Sales)
→ Create → Bins
2
Bin size set karo
Bin size = 500 means har bucket 500 ka range cover karega. Tableau suggest karta hai — ya manually enter karo.
3
Histogram banana
Naya "Sales (bin)" field Data Pane mein aata hai. Ise Columns pe drag karo + Count measure ko Rows pe → Histogram ready!
📊Histogram ka use: "Kitne orders $0-500 ke hain, kitne $500-1000 ke, kitne $1000+ ke?" — distribution samajhne ke liye bins + histogram essential hai!
◈
Hierarchies — Drill-Down Navigation Banao
HIERARCHIES

Sets — Dynamic Subsets Banao
SETS

Sets ek binary classification banate hain — har data point ya toh IN set hai ya OUT of set. Sets dynamic bhi ho sakte hain — jaise "Top 10 Customers by Sales."

FIXED SET

Calculated Fields — Naye Metrics Banao Existing Data Se 🧮
Tableau mein jo field nahi hai woh khud banao — profit margin, shipping days, categories — sab possible
◈
Row-Level vs Aggregate — Sabse Confusing Concept, Clear Karte Hain
FOUNDATION

Calculated Field banana shuru karne se pehle ye samajhna mandatory hai — warna ek common error milegi: "Cannot mix aggregate and non-aggregate arguments."

ROW-LEVEL — Excel Formula Ki Tarah

Aggregation se PEHLE — har row pe individually calculate hoti hai. Excel mein formula ki tarah sochh.

// Shipping Duration calculate karo // Ye formula har row pe apply hoga [Ship Date] - [Order Date] // Profit Ratio per row [Profit] / [Sales]

Tableau is field ko automatically SUM karta hai jab chart pe use karo — pill: SUM(Shipping Days)

AGGREGATE — Pehle Sum, Phir Divide

Aggregation ke BAAD — pehle total nikalo, phir formula apply karo. Financial ratios ke liye.

// Profit Margin (correct way) // Pehle total Profit, phir total Sales // DONO SUM ke andar hain SUM([Profit]) / SUM([Sales]) // Pill: AGG(Profit Margin)

Aggregate calculations mein SUM() agg functions ka use hota hai formula ke andar.

🚨Common Error Fix: "Cannot mix aggregate and non-aggregate" error aayi? Matlab tumne ek formula mein [Profit] (row-level) aur SUM([Sales]) (aggregate) mix kiya. Fix: Ya dono ko row-level rakho ([Profit]/[Sales]) ya dono ko aggregate karo (SUM([Profit])/SUM([Sales])).
◈
4 Types of Calculated Fields — Ek Ek Ko Master Karo
TYPES
1
LOGICAL CALCULATIONS — IF/THEN/ELSE/END
Condition ke basis pe data ko categorize ya flag karo. Maps aur scatter plots mein color-coding ke liye best use case.
// Basic: Profit status classify karo IF [Profit] > 0 THEN "Profitable" ELSEIF [Profit] = 0 THEN "Break Even" ELSE "Loss" END // Advanced: Multiple conditions IF [Discount] > 0.3 AND [Profit] < 0 THEN "High Risk" ELSEIF [Discount] > 0.3 THEN "Discount Alert" ELSE "Normal" END
Use case: Is field ko Color shelf pe drag karo → Map pe instantly green/red/yellow areas dikhengi! Manager ko ek glance mein samajh aajayega.
2
STRING FUNCTIONS — Text Data Clean Aur Manipulate Karo
Text columns pe operations karo — cleaning, parsing, standardizing.
// Inconsistent case fix karo UPPER([Customer Name]) // "john smith" → "JOHN SMITH" LOWER([Product Name]) // "CHAIR" → "chair" PROPER([City]) // "new york" → "New York" // Text search karo CONTAINS([Product Name], "Chair") // TRUE/FALSE return karta hai // Text split karo SPLIT([Customer Name], " ", 1) // First name nikalo SPLIT([Customer Name], " ", 2) // Last name nikalo // Length aur trim LEN([Product Name]) // Character count TRIM([City]) // Extra spaces remove karo
Real use case: "Product Name" mein "Chair-Red", "Chair-Blue", "Table-Oak" hain. SPLIT se type aur color alag karo do columns mein!
3
DATE FUNCTIONS — Timeline Master Karo
Date calculations ke liye — shipping performance, age calculation, trends.
// Shipping days calculate karo (row-level) DATEDIFF('day', [Order Date], [Ship Date]) // Output: 4 (4 din laga shipping mein) // Month ke pehle din pe round karo (time series ke liye) DATETRUNC('month', [Order Date]) // 2023-03-15 → 2023-03-01 // Specific part extract karo YEAR([Order Date]) // 2023 MONTH([Order Date]) // 3 DATEPART('quarter', [Order Date]) // 1,2,3,4 // Future/past dates DATEADD('day', 30, [Order Date]) // 30 din baad
Real use case: DATEDIFF se pata chalega kaunsa region/category ka shipping slowest hai — manager ko report karo!
4
NUMBER FUNCTIONS — Custom Metrics Banao
Math operations, ratios, rounding, ranking sab kuch.
// Profit Margin % (most common metric) SUM([Profit]) / SUM([Sales]) * 100 // Average Order Value SUM([Sales]) / COUNT([Order ID]) // Round karo 2 decimal places pe ROUND(SUM([Profit]) / SUM([Sales]) * 100, 2) // Absolute value (negative ko positive banao) ABS([Profit]) // NULL handle karo IFNULL([Discount], 0) // NULL hai toh 0 use karo ZN([Profit]) // NULL ko Zero mein convert karo
🛠️
Calculated Field Kaise Banate Hain: Analysis menu → Create Calculated Field. Ya Data Pane mein empty area pe right-click → Create Calculated Field. Formula editor mein type karo — Tableau real-time syntax check karta hai, neeche "The calculation is valid" dikhata hai jab sahi ho.

Right-click karo kisi bhi Measure pill pe chart mein → "Quick Table Calculation" → 6 common business calculations instantly available hain:

RUNNING TOTAL
Cumulative sum — har data point pe abtak ka total.

Jan: 1000 → Running: 1000
Feb: 1500 → Running: 2500
Mar: 1200 → Running: 3700
DIFFERENCE

Example: Month-over-Month change in sales.

Jan: 1000, Feb: 1500 → Difference: +500

% DIFFERENCE

East: $200K → Rank 2

LOOKUP Function — Previous/Next Row Dekhna
LOOKUP

LOOKUP ek special table calculation function hai jo ek "time machine" ki tarah kaam karta hai — kisi bhi row ka data peek kar sakte ho.

// Syntax: LOOKUP(expression, offset) // offset = -1 → previous row (pichla value) // offset = +1 → next row (agla value) // offset = 0 → current row (wahi row) // Month-over-Month % Change — Most Used Formula (SUM([Sales]) - LOOKUP(SUM([Sales]), -1)) / LOOKUP(SUM([Sales]), -1) // Previous month ka sales nikalo LOOKUP(SUM([Sales]), -1) // Year-over-Year comparison ke liye SUM([Sales]) / LOOKUP(SUM([Sales]), -1) - 1
💼Financial dashboards mein essential: "Is month ka sales previous month se kitna better/worse tha?" — every finance team needs this! LOOKUP se ye ek formula mein ho jaata hai.

Scope & Direction — Calculation Ko Guide Karo
ADVANCED

Table calculations pe right-click → "Edit Table Calculation" → yahan Addressing (Direction) aur Partitioning (Scope) set hoti hai. Ye concept thoda complex hai — carefully samjho:

ADDRESSING — Direction: Kidhar Chalein?

Calculation kis direction mein move kare?

  • Table Across: Left se right — columns compare karo. Year-over-Year mein use hota hai (har column ek year).
  • Table Down: Upar se neeche — rows compare karo. Ek year ke andar month comparison.
  • Table Across then Down: Dono combine — cross-table calculations.
PARTITIONING — Scope: Kahan Restart Karein?

Calculation kahan restart ho? Kaunsi dimension "wall" banati hai?

Example: Har year ke andar running total restart karna chahte ho. Toh:
• Partition by: Year (har year pe restart)
• Address by: Month (months ke across calculate karo)

Result: Jan-Dec 2022 ka running total, phir restart, Jan-Dec 2023 ka running total.

LOD Expressions — Tableau Ka Most Powerful Feature 🚀
Level of Detail expressions — view se alag granularity pe calculate karo. Ye seekh liya toh Tableau master ho!
◈
LOD Kya Hai — Ek Real Problem Se Samjho
WHY LOD

Problem: Tumhara chart Region level pe hai (4 regions). Tum chahte ho ki har region mein average customer profit dikhao — lekin Customer level pe calculate karke, Region level pe aggregate karo. Regular calculated field ye nahi kar sakti!

LOD expressions tumhe batate hain: "Is calculation ko is specific level pe karo, chahe view mein kuch bhi ho."

FIXED — Always Same Level
{FIXED [Customer] :
SUM([Sales])}

View mein chahe koi bhi dimension ho — yeh HAMESHA Customer level pe calculate hoga. Filters bhi ignore karta hai (dimension filters). Sabse powerful — aur sabse confusing!

Use case: "Pehli order ka amount kya tha?" — FIXED ke bina impossible tha.

INCLUDE — Add a Dimension
{INCLUDE [Product] :
AVG([Sales])}

View ke dimensions ke SAATH extra dimension add karke calculate karo, phir aggregate karo.

Use case: Region level chart pe "Average sales per product per region" — product dimension include karke.

EXCLUDE — Remove a Dimension
{EXCLUDE [Region] :
SUM([Sales])}

View mein Region hai, lekin is calculation mein ignore karo — overall total nikalo.

Use case: "Region ka % of Total Sales" — total nikalne ke liye Region exclude karo, phir divide karo.
